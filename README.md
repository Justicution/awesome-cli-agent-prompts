# awesome-cli-agent-prompts

> A community collection of complete AI Agent Runtime Blueprints.
> Not just prompts — full operational systems with identity, memory, governance, and tooling.

## What is an Agent Runtime Blueprint?

A Blueprint is everything needed to run a consistent, stateful AI agent:

- **Identity** — who the agent is and how it behaves
- **Soul / Principles** — core values and behavioral boundaries
- **Memory Architecture** — how it remembers across sessions
- **Governance Rules** — workspace rules, safety, and scope
- **Tool Configuration** — environment-specific setup
- **Heartbeat / Lifecycle** — periodic self-checks and maintenance

## Submit Your Blueprint

See [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines.

The minimum requirements:
- A `README.md` (philosophy, design notes, author, license)
- At least 2 of the recognized file types (IDENTITY, SOUL, USER, MEMORY, TOOLS, AGENTS, HEARTBEAT, BOOTSTRAP, PRODUCT, PLAN, TASK, CHANGE)

## Design Principles

- Agent is a system, not a prompt
- Structure over language
- Behavior boundaries must be explicit
- All changes must be traceable
- Long-term maintenance is encouraged

## What We Don't Do

- No leaked commercial system prompts
- No generic resource lists
- No single-prompt examples
